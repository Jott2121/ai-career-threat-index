# v2026.3 rebuild — design spec (2026-07-23)

Approved direction: reach & stars; repo becomes the canonical home; mono-repo, static-first.

## What ships

1. **Scoring pipeline** (`pipeline/`, Python stdlib): open rubric + per-factor sub-scores.
   Composite = round(0.45·taskAutomation + 0.25·toolMaturity + 0.20·adoption + 0.10·agenticExposure),
   each factor 0–100. Methodology v2 (v1 was 50/30/20 with no agentic factor) — change documented
   in changelog and METHODOLOGY.md. `pipeline/roles/*.json` are the source of truth; `score.py`
   emits `data/ai-career-threat-index.json` + `.csv` deterministically.
2. **Q3 2026 rescore + expansion to 300 roles.** Existing 76 rescored (history preserved,
   Q3 2026 appended). ~224 new roles across expanded categories (adds Skilled Trades,
   Transportation & Logistics, Hospitality & Service, Science & Engineering, Public Service & Safety,
   Finance-adjacent, etc.). Every role gets a SOC 2018 code and BLS-OES-anchored salary range.
   New roles get history starting Q3 2026 only — no fabricated back-history.
   Two insight tiers: tier 1 (~120 head roles) full insight paragraph + industry modifiers;
   tier 2 tighter 1–2 sentence insight, no industry modifiers.
3. **SOC crosswalk** (`data/soc-crosswalk.csv`): every SOC 2018 detailed occupation → nearest
   scored role slug + match quality (exact | close | approximate).
4. **GitHub Pages explorer** (`site/`, zero-dependency HTML/CSS/JS): search, category/band filters,
   sortable table, risk-vs-salary quadrant chart, per-role trend sparklines, hash-routed
   per-role detail views with shareable links, light/dark.
5. **MCP server** (`mcp/`, Node): stdio MCP server exposing lookup_role, list_roles,
   compare_roles, top_movers, safest_roles, riskiest_roles, crosswalk_soc. Published to npm
   if auth available, else in-repo with install instructions.
6. **Living README**: CI-regenerated SVGs (risk distribution, Q3 top movers, agentic-exposure
   heat strip) + rebuilt copy positioning the repo as canonical.
7. **Quarterly movers report** (`reports/2026-q3-winners-losers.md`), generated by pipeline.
8. **CI**: extended validation tests (sub-score arithmetic, crosswalk integrity, CSV/JSON parity,
   no-fabricated-history rule), SVG freshness check, Pages deploy workflow, tagged release v2026.3.
9. **Ship**: push, enable Pages, GitHub Release (Zenodo picks up new version), sync fresh
   JSON/CSV into meritforgeai-site `public/data/` (static files only — site work stays stopped).

## Schema changes (additive only)

JSON role adds: `socCode`, `tier` (1|2), `subscores` {taskAutomation, toolMaturity, adoption,
agenticExposure}, `agenticRisk` band (Low/Moderate/High/Very High on agenticExposure).
CSV: existing columns unchanged in order; new columns appended after `score_q2_2026`:
`score_q3_2026`, `soc_code`, `tier`, `sub_task_automation`, `sub_tool_maturity`,
`sub_adoption`, `sub_agentic_exposure`.

## Integrity stance

Scores are structured editorial estimates against a published rubric, informed by O*NET task
lists, BLS OES salary data, and public adoption research — stated prominently. No ground-truth
or measurement overclaim. Salary ranges anchored to published BLS OES percentiles (rounded).

## Non-goals

Site rebuild (MeritForge full-stop stands), paid API, non-US labor markets, ML-derived scores.
